# Workflow合规化规范管理方案 - 验证检查清单

## 一、构建打包流程标准化检查
- [ ] CI工作流包含rustfmt代码格式检查步骤
- [ ] CI工作流包含clippy静态分析步骤
- [ ] CI工作流包含单元测试步骤
- [ ] CI工作流包含集成测试步骤
- [ ] CI工作流包含构建验证步骤
- [ ] CI检查失败时阻断CD流程

## 二、版本号生成规则检查
- [ ] 版本号统一从pre-build job输出
- [ ] 支持三种版本号来源：手动输入、Git tag、开发版本
- [ ] 版本号格式符合规范（pro-vX.Y.Z或vX.Y.Z）
- [ ] Docker镜像使用统一版本号
- [ ] Debian包使用统一版本号
- [ ] 所有平台架构使用相同版本号

## 三、Docker镜像标签规范检查
- [ ] 镜像标签符合{image_name}:{version}-{arch}格式
- [ ] 每个架构镜像包含版本标签
- [ ] 每个架构镜像包含latest标签
- [ ] 支持amd64、arm64v8、armv7架构
- [ ] 多架构manifest正确创建

## 四、Docker镜像安全扫描检查
- [ ] 集成Trivy安全扫描工具
- [ ] 配置CRITICAL级别漏洞检查
- [ ] 配置HIGH级别漏洞检查
- [ ] 扫描结果保存为artifact
- [ ] SBOM生成并保存
- [ ] 镜像签名配置完整

## 五、自动触发条件检查
- [ ] push触发限制为main分支
- [ ] tag触发限制为pro-v*和pro-*模式
- [ ] workflow_run触发CI完成后执行CD
- [ ] CI失败时阻断CD流程
- [ ] 环境隔离配置正确
